# Content-brief template (what the tool outputs — NOT finished copy)

When the writer asks for a brief, produce this structure. It tells a human **what to write**;
it never contains the publishable prose itself.

```
## Brief: <cluster / section name>   (priority P<1-3>, gap type: <missing|thin|...>)

WHY THIS MATTERS
- <one line: how many competitors cover it, why it's decision-critical for this page type>

WHO DOES IT BEST (exemplar to study — do NOT copy)
- <competitor brand> — <how they wrote it: structure, what they included, the angle>
- <numbers/examples they used, quoted as facts to verify, not to lift>

WHAT OUR SECTION MUST COVER (outline only)
- H2: <proposed heading>
  - point to cover …
  - point to cover …
  - include: <example type / table / calculator> if competitors do
- Suggested length: ~<N> words (competitor median ≈ <M>)

INTERNAL LINKS TO ADD
- anchor → target topic (and whether we need to build that page)

FAQ TO ADD (questions competitors answer, we don't)
- Q: …

SEO / GUIDELINE CHECKLIST FOR THIS SECTION
- [ ] keyword in H2, [ ] one worked example, [ ] table if comparative,
- [ ] internal links, [ ] schema (FAQ/Product) if applicable, [ ] plain-language readability

NOTE: This is a brief. The writer writes the actual copy.
```
