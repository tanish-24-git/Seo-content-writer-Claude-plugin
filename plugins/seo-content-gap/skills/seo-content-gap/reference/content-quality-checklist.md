# Content-quality & SEO-guideline checklist

Used to answer "are they following guidelines?" and to compute the per-page quality score.
Score each page; report per-brand so OUR gaps are obvious.

## On-page SEO
- [ ] **Title tag** present, ~50–60 chars, includes the primary keyword.
- [ ] **Meta description** present, ~120–160 chars.
- [ ] **Exactly one H1**, containing the primary topic.
- [ ] **Logical heading hierarchy** (no skipped levels; H2 → H3).
- [ ] **Word count** at or above the competitor median for the cluster set.
- [ ] **Internal links** present and relevant (count vs competitor median).
- [ ] **Descriptive anchor text** (not "click here").

## Content depth & helpfulness
- [ ] At least one **worked example / number / illustration**.
- [ ] **Tables** used for comparative or numeric content.
- [ ] **FAQs** present and answer real buyer questions.
- [ ] Covers the **decision-critical clusters** for the page type (sizing, riders, eligibility,
      claims, trust — for a product page).
- [ ] **Plain-language readability** (short sentences, no unexplained jargon).

## Trust / E-E-A-T
- [ ] **Author** named.
- [ ] **Reviewer / expert** named.
- [ ] **Published / last-updated date** shown (freshness).
- [ ] **Trust signals** on page (e.g. settlement ratio, solvency, ratings) where relevant.

## Structured data (schema)
- [ ] **FAQ schema** when an FAQ section exists.
- [ ] **Product / Article / BreadcrumbList** schema as appropriate.

## Compliance hygiene (fintech/insurance)
- [ ] Regulatory/tax claims phrased correctly and **dated** (laws change).
- [ ] No absolute/guaranteed-returns language unless the product warrants it.
- [ ] Disclosures present where benefit claims are made.

> Scoring: quality_score = (items passed ÷ applicable items) × 100, per page.
